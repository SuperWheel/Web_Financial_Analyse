# Spec 010: 巨潮年报批量多年拉取

## ADDED 需求

### 需求：批量多年拉取

系统 SHALL 提供接口，对**一只** A 股证券与**多个**报告年份，依次检索巨潮「年度报告」全文、下载 PDF，并为每份成功下载创建独立的导入任务（不自动 commit）。

#### 场景：全部命中

- **当** 请求体含有效 `q`（或兼容 `code`）与 `years`（1–12 个、1990–2100、去重后升序）
- **且** 每年至少一条全文年报候选
- **则** 每年下载首选候选（检索结果第一条），创建 import job
- **且** 响应中每年 `status=ok`，含 `job_id`、`title`、`pdf_url`、`year`

#### 场景：部分年份无全文

- **当** 某年检索结果为空
- **则** 该年 `status=empty`，`detail` 说明未找到
- **且** 其它年份继续处理

#### 场景：单年下载/解析失败

- **当** 某年下载或建 job 抛错
- **则** 该年 `status=error`，`detail` 含错误信息
- **且** 其它年份继续处理

#### 场景：非法输入

- **当** `years` 为空、超过 12 个、或 `q`/`code` 皆空
- **则** `422` 或 `400`，不发起外网请求

#### 场景：人审约束

- **当** 任一年 `status=ok`
- **则** 对应 job 处于可 review 状态，**不**自动写入 L1 三表

### 需求：前端批量入口

「在线拉取」Tab SHALL 支持年份起止（或等价多年输入），在选定证券后一键批量下载，并展示按年结果（成功可打开核对）。

## 非目标

- 跨多证券的一次请求
- 异步任务持久化 / 进度轮询 API
- 改变单年 009 接口语义
