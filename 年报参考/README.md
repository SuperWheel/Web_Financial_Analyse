# 年报参考样本集

> 用途：冻结 **CAS-simplified-v2** 标准科目与 L0/L1 表结构的设计输入。  
> 原则：精选分层抽样，**不做巨潮批量爬取**；仅收录用户/设计用途的公开年报全文 PDF。

## 目录约定

```text
年报参考/
├── README.md                 ← 本文件
├── samples.csv               ← 机器可读清单与下载状态
├── ashare-cas/               ← A 股 CAS 主集（定 v2 唯一数据源）
├── cross-gaap/               ← 港股 / US GAAP / IFRS 港式对照（不加 CAS 列）
├── hard-cases/               ← 难解析边界（降级策略，不参与加列）
└── optional-financial/       ← 金融对照（不纳入通用 v2）
```

## 命名规范

```text
{代码}_{简称}_{年份}年报.pdf
```

示例：`600519_贵州茅台_2025年报.pdf`

- **优先下载「年度报告」全文 PDF**，不要「摘要」「已取消」版本。
- 年份：优先 **2025**（若尚未披露则用 2024，并在 `samples.csv` 注明）。
- 统计科目时只取 **合并报表** 段，忽略母公司报表。
- **必须是 A 股 CAS 口径**（正文有「合并资产负债表」等），不要 H 股/双语 IFRS「合併財務狀況表」版。

## 下载入口

1. 打开 [巨潮资讯网](https://www.cninfo.com.cn/)
2. 搜索栏输入 **股票代码**
3. 分类选 **年度报告**，找到对应年份 **年度报告** 全文
4. 下载 PDF → 按命名规范放入对应子目录
5. 更新 `samples.csv` 的 `status` 为 `downloaded`

## 定 v2 工作流

```text
1. ashare-cas 满 10 家（A 股 CAS 数字 PDF）  ← 已完成
2. 管道批量：extract → map → 汇总 label / unmapped
3. 产出科目频次表
4. 冻结 CAS-simplified-v2 字段 + rollup
5. 定 L1 扩列 + L0 disclosure_lines
6. 变更包文档 → 再写代码
```

## 统计范围

- **只统计** `ashare-cas/`
- **不统计** `cross-gaap/`、`hard-cases/`、`optional-financial/` 对加列的贡献
- 每份只取 **合并** 三表正文行
